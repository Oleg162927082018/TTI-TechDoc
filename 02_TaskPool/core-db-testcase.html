struct TestCase

Модуль:

test-case.h, test-case.cpp

Назначение:

Структура содержащая данные о тест-кейсе.

QString ID;

    Идентификатор тест-кейса. Этот идентификатор возвращается библиотекой сопряжения при создании тест-кейса и определяет вызовы к какой библиотеке будут выполняться при работе с тест-кейсом.

Сценарии: 1.1 Create test-case.

Пример:

TestCase tc;
tc.ID = "TEST_EMULATOR";


QString fullFileName;

    полный путь к файлу в котором хранятся данные тест-кейса.

Сценарии: 1.1 Create test-case.

Пример:

TestCase tc;
tc.fullFileName = "d:/TTI/tests/example.tcs";


QString currConfigName;

    Имя конфигурации по умолчанию.

Сценарии: 1.1 Create test-case.

Пример:

TestCase tc;
tc.currConfigName = "Default";


QMap<QString, TestCaseConfig> configList;

    Коллекция всех конфигураций.

Сценарии:

Пример:

TestCaseConfig tcCfg;

tcCfg.waitingTime = 600;
tcCfg.maxThreads = 4;
tcCfg.compressionLevel = 2;
tcCfg.isSaveOutput = true;
tcCfg.description = "Test-case example";
tcCfg.extraParams = "<bin>d:/TTI/tests/bin/test-emulator.exe</bin> <prc-bugs>50</prc-bugs> <sources><source id="0">D:/Home/Total_Test_Integrator/bin/test-sorces/Rome</source></sources>";

TestCase tc;
tc.configList.insert("Default", tcCfg);


struct TestCaseConfig

Модуль:

test-case.h, test-case.cpp

Назначение:

Структура содержащая данные об одной из конфигураций тест-кейса.

int waitingTime;

    Время ожидания завершения теста в секундах. Если тест самостоятельно не завершится по истечении этого времени, то он будет прерван средствами операционной системы.

Сценарии: 1.1 Create test-case.

Пример:

TestCaseConfig tcCfg;
tcCfg.waitingTime = 600;


int maxThreads;

    Максимальное количество одновременно запущенных тестов.

Сценарии: 1.1 Create test-case.

Пример:

TestCaseConfig tcCfg;
tcCfg.maxThreads = 4;


int compressionLevel;

    Уровень сжатия базы данных:

  • 0 - без сжатия;
  • 1 - если результат работы теста идентичен текущему бенчмарку, то результат стирается и заменяется ссылкой на бенчмарк;
  • 2 - то же, что и 1 плюс производится сравнение с результатом полученным на предыдущем шаге;
  • 3 - результат работы теста сравнивается со результатами полученными на всех предыдущих запусках.

Сценарии:

Пример:

TestCaseConfig tcCfg;
tcCfg.compressionLevel = 2;


bool isSaveOutput;

    Флаг указывающий сохранять(true) или нет(false) консольный вывод теста.

Сценарии: 1.1 Create test-case.

Пример:

TestCaseConfig tcCfg;
tcCfg.isSaveOutput = true;


QString description;

    Описание тест-кейса.

Сценарии: 1.1 Create test-case.

Пример:

TestCaseConfig tcCfg;
tcCfg.description = "Test-case example.";


QString extraParams;

    Набор параметров тест-кейса, который предоставляется библиотекой сопряжения с тестами и ей же и обрабатывается. Может иметь любой формат представимый в виде строки. Например, набор xml тегов или json выражение.

Сценарии: 1.1 Create test-case.

Пример:

TestCaseConfig tcCfg;
tcCfg.extraParams = "<bin>d:/TTI/tests/bin/test-emulator.exe</bin> <prc-bugs>50</prc-bugs> <sources><source id="0">D:/Home/Total_Test_Integrator/bin/test-sorces/Rome</source></sources>";
