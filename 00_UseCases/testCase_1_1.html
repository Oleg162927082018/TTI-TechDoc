1.1.Создание тест-кейса (Create test-case).

ПОЛЬЗОВАТЕЛЬ: Сразу после запуска приложения может выбирать пункт меню Test-Case -> New

ПРИЛОЖЕНИЕ: Выводит на экран стандартный диалог ввода имени файла в котором будут сохранены параметры тест-кейса.

ПОЛЬЗОВАТЕЛЬ: Вводит имя файла и нажимает кнопку "Ok"

ПРИЛОЖЕНИЕ: Выводит на экран диалог в котором пользователь должен ввести параметры тест-кейсa.

ПОЛЬЗОВАТЕЛЬ: Выбирает тип тест-кейса.
Поскольку тесты могут быть очень разными, иметь разные параметры запуска, разные результаты тестирования и разные алгоритмы сравнения результатов, то логичным было бы иметь для каждого вида тестов библиотеку содержащую необходимые функции. На данном шаге пользователь по сути выбирает эту библиотеку.

ПРИЛОЖЕНИЕ: Выводит описание библиотеки.

ПОЛЬЗОВАТЕЛЬ: Нажимает кнопку добавить конфигурацию, вводит имя конфигурации и нажимает кнопку "Ok".
Наличие нескольких конфигураций необходимо, например, при необходимости запускать тесты на сервере, а анализ проводить на рабочей станции или если тестируемое приложение может быть развернуто на разных стендах и\или под разными операционными системами.

ПРИЛОЖЕНИЕ: Добавляет конфигурацию к списку конфигураций и отображает поля для ввода параметров тест-кейса, которые можно разделить на две гуппы:
Обязательные параметры:

  • Описание тест-кейса;
  • Максимальное время выполнения теста, по истечении которого тест принудительно завершается;
  • Количество одновременно запущенных тестов;
  • Созранять ли консольный вывод;
  • Уровень оптимизации базы данных:
    • без оптимизации;
    • сравнивать с бенчмарком - если результат теста совпадает с бенчмарком, то он удаляется и заменяется ссылкой;
    • сравнивать с бенчмарком и предыдущим - аналогично удаляется, если совпадает с бенчмарком или предыдущим результатом;
    • сравнивать со всеми - аналогично если совпадает с любым из имеющихся результатов, база данных в этом случае будет максимально компактной.

Параметры определяемые библиотекой сопряжения c тестами. Например, это может быть:
  • Путь к исполняемому файлу теста или оболочке выполняющей скрипты, например, bash или cmd;
  • URL сервера на котором развернут тестируемый сервис;
  • Имя пользователя и пароль под которыми происходит авторизация;
  • Список папок или список файлов в которых находятся конфигурации или скрипты тестов;
  • и т.д.
